# Kinodel LangGraph Runtime

## Core model

Kinodel runtime compiles a frozen `pipeline_spec` into a LangGraph `StateGraph`.

```text
PipelineRegistry -> pipeline_spec.json -> GraphRuntime -> StateGraph -> services -> artifacts
```

Skills are no longer the state machine. They are role contracts used by graph nodes.

## Required LangGraph primitives

- **`StateGraph`** — compile stage specs into nodes and edges.
- **Checkpointer** — durable project/thread pause-resume.
- **Thread ID** — one durable runtime thread per active project run.
- **`interrupt()`** — BriefGate and ReviewGate cards.
- **`Command(resume=...)`** — user decision resume.
- **Conditional edges** — gate routes, repair loops, stop/pause.
- **Streaming/events** — progress and previews to Open WebUI and future dashboards.
- **Subgraphs** — specialist agents can become isolated subgraphs later.

## Compact state

```json
{
  "project_id": "demo",
  "project_dir": "/projects/demo/v1",
  "pipeline_id": "cinematic.v1",
  "current_stage": "p7_story_images_gate",
  "pending_gate": {"gate_id": "p7"},
  "artifacts": {
    "brief": "brief.json",
    "story": "story.json"
  },
  "selected_media": {
    "story_frames": [
      {"path": "outputs/shot_01.png", "url": "https://..."}
    ]
  },
  "gate_decisions": [],
  "chunk_refs": [],
  "last_error": null
}
```

Do not store full artifact bodies, chat transcripts, provider payloads, raw logs, vector hits, or media blobs in graph state.

## Node patterns

### BriefGate node

Builds a visible brief card from user input and defaults, then interrupts. On approval, ProjectStore creates the project and writes `brief.json` plus frozen `pipeline_spec.json`.

### Agent stage node

Receives a typed handoff:

- project ID and directory
- declared reads
- declared writes
- selected media refs
- selected chunk/context-pack refs
- optional edit notes

The node calls the owner skill/agent, validates the produced artifact, and advances.

### Render stage node

Consumes only complete render request artifacts. It submits jobs through RenderService, persists provider scratch outside graph state, streams progress, promotes compact result manifests, and resumes the graph.

### ReviewGate node

Creates a card:

```json
{
  "type": "review_gate",
  "gate_id": "p7",
  "label": "Story Images ReviewGate",
  "preview_refs": [],
  "choices": ["A", "B", "C", "D"]
}
```

Resume routes:

- `A` -> next production stage
- `B` -> critic loop
- `C` -> edit loop with notes
- `D` -> pause

### Montage node

Calls MontageService with selected video refs. It validates final MP4 existence/readability and advances.

### Chunk node

Calls ChunkService/Craft after final approval. It writes final memory/chunks from approved artifacts only.

## Service boundary

| Service | Owns |
| --- | --- |
| `PipelineRegistry` | Active specs, schema validation, project-local freeze. |
| `GraphRuntime` | Graph compilation, thread/checkpoint, edge execution, interrupts. |
| `ProjectStore` | Layout, atomic writes, artifact refs, public media refs. |
| `GateService` | Gate cards, decision parsing, repair routing. |
| `AgentService` | Specialist execution adapters and subgraphs. |
| `RenderService` | Render jobs, provider adapters, scratch, result promotion. |
| `ProviderRegistry` | Capability binding for ComfyUI, fal.ai, hosted providers. |
| `MontageService` | Final assembly. |
| `ChunkService` | Final chunks, context packs, indexing. |
| `NotificationService` | Streaming progress, media previews, UI events. |

## Repair loops

Repair is graph behavior, not free-form Producer improvisation.

- Gate `B` calls `critic-kinodel`, writes scoped QC notes, and routes to the owner of the affected artifact.
- Gate `C` sends user notes directly to the owner of the affected artifact.
- Repaired artifacts must overwrite only their owned artifact and pass validators.
- Downstream artifacts invalidated by a repair must be regenerated by graph edges, not patched by chat prose.

## Render wake-up replacement

Legacy terminal continuation chains become backend events:

```text
render node submits job
-> RenderService persists job/run IDs
-> provider adapter updates job state
-> result manifest is promoted
-> GraphRuntime resumes the thread
-> NotificationService emits preview/gate event
```

No stage should require a bespoke wake-up script.

## Custom pipelines

A new pipeline should add a spec and validators, not clone Producer prose. Required spec fields:

- stage `id`
- `type`
- owner/service
- `reads`
- `writes`
- validator
- edge or routes
- selected refs source when needed
- gate policy when human decision is required
