# Token Aerodynamics

Use this contract to keep Producer and subagent context small and cache-friendly.

## Core rule

Stable prefix first, dynamic suffix last:

1. Stable prefix: skill law + static delegate contract.
2. Dynamic suffix: one compact handoff JSON.
3. Volatile scratch: files or `/tmp/kinodel/...`, never prompt text unless debugging.

Importance does not require first-token placement. `stage.goal` and `stage.owner_skill` are authoritative, but they live inside the final handoff envelope so repeated delegate prompts share the same prefix.

## Do pass

Pass only path/ref data needed by the current owner:

- `project.id`
- `project.dir`
- `artifacts.read` relative paths
- `artifacts.write` relative path
- `stage.goal`
- `stage.owner_skill`
- selected media refs from `render_results/*.json.selected_outputs`
- tiny `context_cache` entries generated by `state_guard.py` from hot artifacts (`brief.json`, `story.json`): `path`, short `sha256`, and small summaries only
- user edit notes for the current gate only, in `edit_notes`

## Do not pass

- full JSON bodies unless a human preview needs a short excerpt
- provider queue responses
- render logs
- retry scratch
- unrelated QC notes
- completed-stage history
- everything in `outputs/` except selected path refs already recorded in result manifests
- repeated static constraints as runtime JSON

## Canonical handoff shape

`state_guard.py handoff` should emit this nested shape. Stable fields appear first; high-volatility media URLs and edit notes appear last.

```json
{
  "schema": "kinodel.delegate_handoff.v1",
  "contract": "producer.delegated_design_stage.v1",
  "project": {
    "id": "demo",
    "dir": "/home/user/projects/demo/v1"
  },
  "artifacts": {
    "read": ["brief.json", "story.json", "render_results/main_frame_result.json"],
    "write": "storyboard_requests.json"
  },
  "stage": {
    "goal": "p5_storyboard_plan",
    "owner_skill": "storyboard-kinodel"
  },
  "selected_media": [
    {"shot_id": "main_frame", "path": "outputs/main_frame.png", "url": "https://..."}
  ],
  "context_cache": [
    {"path": "brief.json", "sha256": "16hex", "summary": {"shot_count": 3, "aspect_ratio": "1:1"}},
    {"path": "story.json", "sha256": "16hex", "summary": {"scene_count": 3, "story_excerpt": "..."}}
  ],
  "edit_notes": null
}
```

Static constraints belong in the delegate contract, not repeated in the handoff JSON:

- read the whole handoff before acting
- load/follow exactly `handoff.stage.owner_skill`
- execute exactly `handoff.stage.goal`
- read only `handoff.artifacts.read` under `handoff.project.dir`
- write exactly `handoff.artifacts.write`
- use `handoff.context_cache` only as a compact hint; authoritative content remains on disk
- preserve `handoff.project.id`
- set `status=complete`
- avoid provider runtime fields
- return only `{status, artifact_path, summary}`

## `context_cache` semantics

`context_cache` is not a cross-conversation memory store and is not a second source of truth. It is recomputed on every `state_guard.py handoff` call from files already listed in `artifacts.read`.

Why keep it if `artifacts.read` already exists:

- `artifacts.read` tells the delegate what to open.
- `context_cache` gives the delegate a tiny orientation/checksum before opening files.
- `sha256` helps detect “I read a different file than the Producer meant”.
- Summaries make handoff/debug output human-readable without pasting full `brief.json`/`story.json`.

Duplication budget: only tiny stable facts are allowed (`shot_count`, `aspect_ratio`, `scene_count`, short excerpt). If a value is long, volatile, provider-specific, or required for correctness, it belongs only in the artifact file.

## Cache-input token strategy

Current Hermes/LLM prompt caching benefits stable prefixes. Make repeated stage prompts share a stable, small prefix:

1. Load stable skills/references first: `pipeline-kinodel`, then this Producer skill.
2. Do not load designer specialist skills into Producer context during normal production. Use `delegate_task` with a stable static contract and one compact handoff JSON; let the subagent load/follow its specialist skill and write the artifact on disk.
3. Keep `delegate_task.goal` stage-neutral: do not include `p5_storyboard_plan`, artifact names, or specialist names in the goal string.
4. Keep the top of `delegate_task.context` stage-neutral. Put all project/stage/edit data only in the final handoff JSON block.
5. Keep `SKILL.md` short and stable; put volatile examples/provider payloads into references/scripts.
6. Put changing project state in files, not chat. Pass file paths and compact selected refs.
7. Keep long creative artifacts out of Producer context. Specialists read files directly.
8. Use `render_results/*.json.selected_outputs` as compact media manifests instead of rescanning `outputs/`.
9. Put provider scratch under `/tmp/kinodel/<project_id>/<run_id>/`; never include it in delegate prompts.
10. For repeated subagent calls, reuse the same static contract text and handoff schema so only the final JSON suffix changes.

## Producer read budget

Producer may inspect artifacts, but must not internalize them. Normal reads are limited to:

- schema/status/project_id for validation
- `jobs` shape for render request artifacts
- `selected_outputs` for media chaining and gates
- short story/preview fields for ReviewGates
- minimal refs/summary fields for `final_chunk.json`

Everything else stays on disk and is read deeply only by the current owner specialist.

Practical rule: if a blob changes every run or is only useful for debugging, it is L6 scratch and should not enter the prompt unless diagnosing a failure.
