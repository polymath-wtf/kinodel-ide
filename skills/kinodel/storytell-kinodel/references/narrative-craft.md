# Storytell Narrative Craft Reference

Keep this small. The owner skill remains authoritative; this reference is only a taste checklist for better `story.json` artifacts.

## The Kinodel micro-film test
A good Kinodel story should answer these in one glance:

1. Who/what is the visual anchor?
2. What pressure or desire makes the frame alive?
3. What changes across the shots?
4. What is the one image the user will remember?
5. What emotion remains at the end?

## Compact arc by shot count
- 3 shots: specific setup → escalation/reveal → iconic payoff.
- 4 shots: setup → problem → turn → payoff.
- 5 shots: setup → tension → escalation → climax → aftermath.
- 6 shots: character/world → problem → oh-crap → turn → aha/climax → after-image.

## Anti-boring rules
- Do not write “a cat walks into a room.” Write the room, the angle, the light, the strange object, and why this moment matters.
- Do not make all shots the same distance. Alternate wide / medium / close / detail.
- Do not hide the story in abstract nouns. Use objects and actions: broken neon sign, spilled milk, shaking hand, rain on velvet, cracked phone screen.
- Do not rely on downstream agents to invent stakes. Give them a visual beat with tension.

## Fields downstream agents love
`beat`, `emotion`, `composition`, `visual_vibe`, and `transition_to_next` are optional but useful. They are compact and prevent later prompt agents from guessing the cinematic intent.
